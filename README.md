# zhamel-zfskey-kmod

`zhamel-zfskey-kmod` is the FreeBSD kernel module that consumes the
`zhamel_zfs_key` preload payload prepared by `zhamel`, imports the root pool if
needed, and loads the native OpenZFS wrapping key into the target encryption
root during early boot.

The module installs as:

```text
/boot/modules/zhamel_zfskey.ko
```

## Scope

This repository only contains the kernel module source and release automation.
It is intentionally separate from:

- `zhamel`, the EFI loader

The FreeBSD package/port metadata lives outside this repository in the
`foji-bsd` ports tree.

## Build

Build on FreeBSD against a source tree matching the target kernel/OpenZFS
version:

```sh
make SYSDIR=/path/to/freebsd-src/sys
```

Typical local example:

```sh
make SYSDIR=$HOME/devel/system/freebsd-src/sys
```

Install on a test machine:

```sh
doas make SYSDIR=/path/to/freebsd-src/sys install
```

The installed module is loaded by the FreeBSD boot path after `zhamel` has
preloaded the payload. Manual test loading is possible with:

```sh
doas kldload /boot/modules/zhamel_zfskey.ko
```

## Notes

- The module depends on the in-tree OpenZFS headers from the target FreeBSD
  source tree.
- The source tree used at build time should match the kernel/OpenZFS version
  you plan to run.
- The loader is expected to preload the binary payload under the
  `zhamel_zfs_key` module info type before kernel handoff.
- The payload starts with `struct zhamel_zfskey_header`, followed by the
  NUL-terminated ZFS encryption root name and the 32-byte raw OpenZFS wrapping
  key.
- The module wipes the preloaded payload after processing it.

## Release Artifacts

Tagged releases on `main` produce Forgejo release assets through the
repository's Forgejo runner workflow:

```text
zhamel-zfskey-kmod-<tag>.tar.gz
zhamel-zfskey-kmod-<tag>.tar.gz.sha256
zhamel-zfskey-kmod-<tag>.tar.gz.sig
```

The signature is generated by the Forge organization signing secret consumed by
the source-release workflow.
